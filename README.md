# FrameModelMaker-Web

**Version 1.1.0**

FrameModelMaker-Web は、立体フレーム解析モデルをブラウザ上で作成・検証・可視化し、JSON / YAML で受け渡すための TypeScript アプリケーションです。インストール不要で、節点・部材・断面・材料・境界条件・バネ・壁・荷重を表形式と 3D ビューから編集できます。

> C# Windows Forms アプリケーション「Abi-FrameTerminal」のモデル編集機能を、Web 向けに再設計したプロジェクトです。本アプリ自身は FEM 解析を実行しません。

## 主な機能

- モデル診断基盤: 重複番号、参照切れ、ゼロ長部材、不正な物性・荷重配列などを ERROR / WARN / INFO で診断
- 編集履歴: Undo / Redo、未保存表示、復旧用スナップショット、参照追従付き再採番、重複節点統合
- 連携選択: 3D の節点・部材・壁とデータグリッドを双方向に選択
- 表編集グリッド: 数値・列挙・参照等の型付きセル、複数行選択、検索、表示ソート、TSV コピー＆ペースト、CSV 出力
- 3D / 2D 表示: 標準視点、透視 / 平行投影、9レイヤー、番号、色分け・凡例、平面/立面での作図・移動・複製
- 荷重・境界条件表示: 節点荷重と6自由度境界条件の 3D グリフ表示。部材荷重表示は実験的です
- 荷重ケース: 名前・種別・メモ、追加・複製・削除と荷重組合せ
- ファイル交換: バージョン付き JSON、旧 JSON の移行、strict / lenient 読込、解析用 YAML の診断付き入出力
- 断面・数量: 代表断面の断面性能計算とモデル統計・数量集計の基盤
- 解析結果ビュー（実験的）: 外部 JSON の変位・反力・部材端力を読み込み、変形表示と時刻歴アニメーション
- 日本語 / 英語、ダーク / ライトテーマ、キーボード操作、レスポンシブレイアウト

## 基本操作

### ファイル

- **新規作成**: 空のドキュメントへ切り替えます。未保存変更がある場合は確認します。
- **開く**: `.json`、`.yaml`、`.yml` を読み込みます。入力サイズの上限は 25 MiB です。
- **保存**: 現在の作業モデルを JSON v2 で保存します。検証エラーは診断で確認できます。
- **YAML 出力**: YAML 由来の解析メタデータを可能な範囲で保持して再出力し、表現できない項目を警告します。
- **CSV 出力**: 現在のタブを表形式で出力します。
- **サンプル読込**: 同梱モデルを読み込みます。

読み込みは一時ドキュメントで完了してから置換されます。未知の将来 JSON バージョンや不正ファイルで、編集中のモデルを部分的に上書きしません。

復旧スナップショットは現在 `localStorage` へベストエフォートで保存します。ブラウザの保存容量を超えた大きなモデルでは復旧データを保存できないため、JSON ファイルもこまめに保存してください。IndexedDB を使う大容量向け復旧は未実装です。

### データ編集

右側のタブで、節点、部材、断面、材料、境界条件、バネ、壁、節点荷重、部材荷重、CMQ 荷重を編集します。

- 行クリックで選択、`Ctrl` / `Shift` で複数選択
- 検索欄で表示行を絞り込み
- `Ctrl+C` / `Ctrl+V` で選択範囲を TSV としてコピー＆ペースト
- 参照列は存在する番号から選択
- 番号変更、再採番、節点統合は参照先も同一操作で更新
- `Ctrl+Z` / `Ctrl+Y` で Undo / Redo
- `Delete` で選択行を削除。参照中の要素は安全確認の対象

荷重タブは上部の荷重ケース選択と連動します。荷重ケースは最低 1 件を維持し、ケース ID を使って荷重組合せと関連付けます。

### 3D ビュー

| 操作 | 動作 |
|---|---|
| 左ドラッグ | 回転 |
| 右ドラッグ | 平行移動 |
| ホイール | ズーム |
| 要素クリック | 節点・部材・壁を選択 |
| キャンバス上の矢印キー | キーボード照準を移動 |
| キャンバス上の `Enter` | 照準位置で選択または作図を確定 |
| `F` | モデル全体を表示 |
| `Esc` | 選択または作図モードを解除 |

ツールバーから上面・正面・側面・アイソメ、透視 / 平行投影、2D 平面・X/Y立面、節点・部材作図、選択要素の移動・オフセット複製、荷重、解析結果を切り替えられます。番号と境界条件の表示は「表示」メニュー、全レイヤー・選択のみ表示・色分け・凡例・ラベル密度は「モデル情報」から変更できます。非操作時は再描画を停止します。部材・壁の移動は構成節点を移動するため、共有節点に接続する他要素も追従します。

### 検証とツール

- **モデル検証**: 診断の重要度、コード、対象番号、説明を表示
- **モデル情報**: 要素件数、部材総延長・体積、孤立節点を表示し、断面別数量を CSV 出力
- **断面計算**: 矩形、円形、箱形、I / H 形の寸法から `A / J / Iy / Iz` を算定
- **インポートレポート**: YAML 読込後に単位変換、スキップ要素、保持・欠落情報を確認。読込前プレビューとレポートファイル出力は未実装
- **解析結果（実験的）**: 検証済みの外部解析結果 JSON を読み込み、変形・反力・断面力の表示と倍率、フレームを操作。モデル編集時は古い結果を自動的に破棄

## ファイル形式

### Frame JSON v2

保存 JSON には `formatVersion: 2` が必ず付きます。バージョンなしの従来ファイルは v1 として移行されます。v2 では、従来のモデル要素に加えて次を保持します。

- 安定 ID を持つ `loadCases`
- `loadCombinations`
- YAML 解析情報を保持する `analysisMetadata`
- 断面の正式なねじり定数 `torsionConstant`。旧 `p2_Ix` は互換フィールドとして保持

`parseFrameJson()` は `strict` と `lenient` を選べます。strict はパーサーが検出した型違反を拒否し、lenient は補正内容を診断として返します。画面からのJSON読込はstrictを先に試し、旧ツールや手編集ファイルの型違反だけを診断付きlenient読込へフォールバックします。未知の将来バージョンは拒否します。JSON Schema による完全検証ではないため、読込後と検証済み出力前にはモデル診断も実行してください。

### 解析用 YAML

`schema_version: "1"`、`units`、`model` を持つ YAML を扱います。入力単位 `mm / N` 系は内部の `cm / kN` 系へ変換されます。節点、支持、材料、断面、主要線材に加え、荷重ケース・荷重組合せと次の情報を `analysisMetadata` に保持します。

- `equalDOF` 制約
- 節点質量
- `twoNodeLink3D` の剛性・方向
- グループ、抽出指定、元タグなどのトレーサビリティ

非対応・簡略化・スキップした要素は診断へ記録されます。再出力は節点、支持、材料、断面、線材、解析メタデータ、荷重ケース・組合せに対応します。現行解析 YAML スキーマで表現できない壁、部材端バネ、FrameModelMaker 固有の節点 / 部材 / CMQ 荷重は出力診断で警告します。YAML の全スキーマを損失なく往復できることを保証するものではありません。

### 解析結果 JSON v1

解析結果はモデル JSON と別ファイルです。`formatVersion: 1`、任意の `loadCaseId` / `combinationId`、時刻順の `frames` を持ちます。各フレームには節点変位・回転・反力と、部材 I/J 端力を格納できます。静的な単一フレーム形式も読み込めます。

> **実験的機能:** v1 は `units.length = "cm"`、`units.force = "kN"`、`units.moment = "kN-cm"`、`coordinateSystem = "global-xyz"`、`nodeReactionSystem = "global-xyz"`、`memberForceSystem = "local-xyz"` を必須とします。欠落・不一致、現在のモデルにない節点・部材・荷重ケース・組合せの参照は読込時に拒否し、ビューアは単位・座標・局所軸・符号を変換しません。モデル編集時は読み込み済みの結果を破棄します。外部結果はこの規約へ変換・検証してから読み込んでください。断面力図は確認用の実験的表現であり、この表示だけを構造安全性の判定に使用しないでください。

## 単位系と座標系

| 種別 | 内部単位 |
|---|---|
| 長さ | cm |
| 力 | kN |
| モーメント | kN·cm |
| 応力度・弾性係数 | kN/cm² |
| 断面積 | cm² |
| 断面二次モーメント・ねじり定数 | cm⁴ |

座標系は X = 水平右、Y = 水平奥、Z = 鉛直上です。外部形式との単位・局所軸・符号規約は、インポート診断と [技術仕様書](docs/SPECIFICATION.md) を確認してください。

## 実装上の制約

- FEM ソルバーは含みません。解析結果は外部ソルバーから読み込みます。
- 復旧履歴は上限付き JSON スナップショットを `localStorage` に保存するベストエフォート方式です。IndexedDB、差分履歴、大容量モデルの確実な自動復旧は未実装です。
- YAML 再出力は保持できた情報を対象とするベストエフォートです。
- 壁厚の正式仕様がないため、壁体積・重量は数量集計の対象外です。
- `Material.unitLoad` の意味を指定しない限り、材料重量は確定値として扱いません。
- 大規模モデル向けにイベント駆動描画とラベル間引きを実装していますが、Web Worker と完全なグリッド仮想化は今後の拡張です。
- 実験的な解析結果表示は上記の固定単位・座標規約だけを受け付け、変換は行いません。節点・部材番号と荷重参照は現在のモデルへ照合しますが、値、部材局所軸と符号の妥当性は入力元で確認が必要です。

## プロジェクト構成

```text
src/
  app/          ダイアログ、ツールパネル、ファイル出力
  io/           JSON v2・解析 YAML の入出力
  models/       フレームモデル、荷重ケース、解析メタデータ・結果
  services/     履歴、断面性能、モデル統計
  validation/   モデル健全性検証
  ui/           型付きデータグリッドとクリップボード処理
  viewer/       Three.js ビューア、2D 作図、結果表示
tests/          モデル、IO、サービス、UI、ビューアのテスト
docs/           技術仕様と設計監査
```

詳細は [技術仕様書](docs/SPECIFICATION.md) と [追加機能提案・実装記録](docs/FEATURE_RECOMMENDATIONS.md) を参照してください。
